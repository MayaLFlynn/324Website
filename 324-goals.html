Development

  • Medium-scale abstraction: use the principles and patterns of abstraction to design elegant, extensible, and maintainable software modules for medium-scale (multi-component) systems.
  • Medium-scale architecture: use common software architecture patterns to build medium-scale (multi-component) systems.
  • Domain-specific architecture: use knowledge of a particular computing domain to build medium- to large-scale systems within that domain.

Verification

  • Testing dimensions: employ fundamental testing strategies for designing software tests that adequately cover a medium-scale program.
  • Testing infrastructure: implement software tests that are fully integrated into a development pipeline, including development and deployment.

Design

  • Needfinding
  • Prototyping
  • Evaluation
  • Accessible design: design software systems that account for a variety of user abilities and disabilities.

Engineering

  • Infrastructure
  • Self-Learning
  • Process
  • Collaboration
  • Quality: develop artifacts (both software and otherwise) that exhibit completeness, thoroughness, and polish.
