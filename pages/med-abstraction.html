  • Factory class

    • I designed the prestige button and its functionality. Like all of the other interactable items, it has two components, the UI component and the functionality component. I designed both of the classes for these. In the UI component, we have three different functions. A function that allows you to create a prestige object, one that defines behavior when the mouse is hovering over the object and one that defines behavior when it is clicked. These functions are also in other classes, so that in elsewhere in the program, we can loop over each object and do those functions if they are needed.

    When the UI prestige button is created, it will be linked to the functionality piece of the prestige button. This has only two functions, mimicing other Buyable items in our program. The functionality of each button in our game happens within an action called 'buy' which checks whether the conditions have been met in order to purchace the item, and then does that purchacing. In the prestige button case, 'buy' means do the action prestige which is to reset the game and double the global multiplier. This function allows to hide the direct functionality of the button, and simply call it like we would any other button. If we ever wanted to change the functionality of prestige, such as one of the several ways significantly more complex ways we envisioned it at the beggining of our project, we would only have to change what happened in this one funciton.
