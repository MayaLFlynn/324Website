Quality

Quality: Provide an overall evaluation of the overall quality of your codebase.
In what ways does your codebase exemplify engineering excellence?
In what ways does it fall short? How would you addressed these issues if given more time?

Our PaperQuest Game and the code underlying it is high quality. The code is architected well, and abstracted logically. Since none of the code was written by AI, we were able to take full control of the understanding and seperation of concerns. Each time that we have had to change something in our code, it is easy to find where it needs to be changed, and make the changes without disrupted anything. The functions are documented, which provide inline warnings when the parameters or return values do not match the requirements of the documentation. We have constantly ensured that the main branch in GitHub is clean, and that every push to the main branch is seen and approved by at least one other group member.

Our code of course cannot be perfect. One of the main ways that it falls short is through its testing suite. Love2D doesn't provide an easy framework for code to be tested, so our code could have more comprehensive tests, especially in the UI pieces of our code. With more time, we may be able to find or build a framework that allows us to test what is visible on the screen at what time. In addition, there are many features of our game that we were not able to realize such as achivements and mini-boss fights, that we would implement with more time. However, in the end, we still believe that our final product is fun to play, and demonstrates engineering excellence.
