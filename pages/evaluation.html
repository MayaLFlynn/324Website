Evaluation

Identify relevant artifacts from the human-centered design process that address the following points.
What was the design of your user study to evaluate your artifact?
What were the results of this user study?
What were the main points of future development that you took away from this activity?

The artifact that I was asking users to evaluate was the code at the state from April 15, 2026. This was just after the commit 'Fixed testing mode so tests reset upon closing/opening the menu'. The user evaluation was to determine if the mechanics felt right, if the UI was intuitive, and if the game was enjoyable. This was done before modifing the code to fit with user design principles so that we could have a better sense of what needed to be changed, and what real users found intuitive or confusing. I set up a testing environment, and asked each of the users the same questions as I watched them play the game.

From the interviews I did I found that several aspects of our game were confusing to users. The first and biggest was that the quanity of paper and wood along with the paper and wood per second were confusing. One user kept trying to hover over them to determine if there was information about them, and the other just played the game without knowing and commented that they were confused. They both would have liked more information about what the weapons get the user as well as the wood to paper conversion.

In our next UI discussion and implementation, we need to have a clear distinction between boxes that provide information and boxes that are interactable. Having a hover function over both interactable buttons and informational areas may be useful. In addition, it may be useful to have the mouse change when hovering over something interactable. There may also need to be a layout change so that the information is in one place on the screen towards the edge, and the buttons are in another more central location.
