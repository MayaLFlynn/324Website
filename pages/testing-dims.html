Testing Strategies

Identify a component of your project that you were responsible for testing.
According to code coverage tools, what aspects of this component are covered with tests?
What kinds of tests are they?
For code that is not covered by tests, why did you not cover them?

I was responsible for creating and testing the Math library for our game. It was designed in order to allow precision when numbers pass Lua's 2^53 promised number. Every component of every function in the mathHander file was tested with the tests. In addition, every branch of create new BigNum in big_number was also tested through the test suite. These tests are static. They test whether the functions in the math library work or not for the given inputs. I would have liked have the program choose different numbers each time, but I wouldn't have been able to test thier legitamacy. The tests are unit tests.

The code that is not directly covered by tests includes the Object class file. This code was adopted by Aubrey for our use case, from rxi's code on GitHub. Since the code simply allows us to create an Object, it gets indirectly tested whenever a BigNum is created. Also, the tests for power and division haven't been written, as power and division haven't been implemented. The tests for multiplication and modulo have been written, but are not called on every run because the implementations for multiplication and module have not been completed.
